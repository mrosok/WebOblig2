Scalable Vector Graphics

SVG er et bildeformat. Det vesentlige som skiller SVG fra andre bildeformater på nettet (JPG, GIF, PNG etc) er at det er vektorbasert, i stedet for å være bitmap/pixel-basert. Bitmap-bilder består av mange bits med forskjellige fargekoder som tilsammen utgjør et bilde. Når vi forstørrer bildet kan vi se de individuelle bitene. Bitmap-bilder tar forholdsvis stor plass og egner seg godt til digitale foto med komplisert farge-gjengiving, men skalerer dårlig oppover.

Vektor-grafikk er punkter som henger sammen av linjer eller såkalte Bézier-kurver. Det er matematiske definisjoner som beskriver forholdet mellom punktene. Det gjør at bildet kan lett skaleres opp og ned uten at det går ut over oppløsningen til bildet. Vektorgrafikk tar også mye mindre plass en bitmap-grafikk, men egner seg dårlig for fotografier og lignende kompliserte bilder.

SVG er vektorbasert grafikk for websider. Det bruker XML for å definere grafikken. Det betyr at vi i tillegg til å referere til en SVG fil i html-dokumentet, også kan skrive SVG-grafikk direkte i HTML. Vi kan således manipulere bildene, f.eks gjennom animasjon.

Eksempel på SVG

Kode:

<svg width="200" height="200">
	<circle cx="100" cy="100" r="70" stroke="red" stroke-width="4" 
	fill="black" />
</svg> 
<svg width="400" height="200">
	<rect x="10" y="10" height="80" width="120" stroke="green" 
	stroke-width="2" fill="lightblue"/>
	<polyline points="90,10 400,180 90,180" fill="red" />
</svg>
<svg id="line-rotation" width="100" height="100">
	<line x1="0" y1="0" x2="100" y2="100" stroke="purple" stroke-width="5"/>
</svg>

#line-rotation {
	animation-name: rotate;
	animation-duration: 10s;
	animation-iteration-count: infinite;
	transform-origin: 50% 50%;
}

@keyframes rotate {
	0% { transform: rotate(0deg); }
	100% {transform: rotate(360deg);}
}
