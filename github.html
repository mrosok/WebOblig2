GitHub Versjonkontroll er et system for organisering av endringer i dokumenter, dataprogrammer, websider og andre prosjekter med mye datainnhold. Hver endring utgjør en ny "revisjon" av prosjektet, og endringen stemples med et tidspunkt og signatur av personen som utførte endringen. Det å kunne sammenligne endringer, gå tilbake til tidligere revisjoner og lett se arbeidsflyten i prosjektet er helt nødvendig når prosjektene begynner å bli store, og særlig når flere mennesker jobber sammen. GitHub er et versjonkontrollsystem, men også en hosting-service, suplementert med litt sosiale medier. GitHub lar brukere laste opp prosjekter som brukere kan jobbe sammen om. Det er mest brukt for programkode, men kan lett brukes for alle mulige filtyper. GitHub har mange funksjoner som skiller seg ut fra andre versjonkontrollsystemer. En av de mest nyttige er "forking"- en avgrening/kopi av et prosjekt som kan kopieres fra en brukerkonto til en annen. Med forking kan du ta et prosjekt du ikke har direkte skriverettigheter til og kopiere det til deg selv. Du kan så gjøre endringer lokalt og hvis du ønsker å bidra med dine endringer kan du sende en "pull request" til eieren av prosjektet. Hvis eieren godtar endringene dine kan han "merge" endringene inn i prosjektet. Fork, pull request og merge er utrolig nyttig i open source prosjekter. Open source er også en del av kjernen i GitHub. Man kan selvsagt jobbe med private, lukkede prosjekter, men det at GitHub har tilrettelagt så mye for kommunikasjon gjør at open source utvikling er veldig naturlig og enkelt. Det er over 24 millioner utviklere på GitHub, og alle har mulighet til å bidra til prosjekter som er open source. Med wiki-sider, "follow" av personer og andre funksjoner er GitHub nesten som facebook for kodere.
