Git og GitHub

Git er et versjonskontrollsystem, det vil si et system som holder styr på alle versjoner og iterasjoner et prosjekt har gått igjennom. Det er et utrolig nyttig verktøy for store prosjekter og spesielt når flere personer jobber med det.

Med Git har vi mulighet til å se alle endringene som er blitt gjort i et prosjekt, og hvem som har gjort de. Vi kan hente inn tidligere versjoner om vi er misfornøyd med endringene. Slik fungerer Git også som en backupløsning. Vi kan «branche» prosjektet ut og jobbe med en lokal kopi, for så å «pushe» endringene tilbake i den endelige versjonen. Om flere jobber og gjør forskjellige endringer på samme sted i en fil, vil man få mulighet til å kontrollere konflikten og velge hva man skal ta vare på. En slik kontroll over arbeidsflyten i et prosjekt er uvurderlig.

Git har mange nyttige funksjoner utover det som er standard i versjonskontrollsystemer. En av de kraftigste er det som kalles en «pull request», som man kan utføre hvis man ikke har direkte skriverettigheter til et prosjekt. I et open source miljø er dette svært anvendelig. Sett at en gruppe mennesker sitter med hovedansvaret for prosjektet og kan skrive/endre på det, men alle andre har mulighet til å se på koden, laste den ned og gjøre endringer lokalt. Da kan hvilken som helst utvikler bidra ved å gjøre endringer og forslag lokalt og sende en «pull request» til de som har ansvaret. Disse kan gå igjennom alle forslagene og velge om de vil «merge» endringene inn i prosjektet.

Git i sin reneste form brukes gjennom et terminal-vindu hvor man gir kommandoer, men det finnes GUI-klienter som er lettere å bruke, men kan være mindre fleksible. Noen eksempler på populære Git-klienter for Windows er GitKraken, SmartGiT og SourceTree.

GitHub baserer seg på Git-teknologi, men er i tillegg til versjonskontrollsystemet også en hosting service, slik at man kan lagre prosjekter på deres servere. Det er også et samfunn av utviklere som jobber sammen, og har mange funksjoner for kommunikasjon mellom brukere. GitHub er en god løsning for å jobbe med kode i et team. Alle kan ha tilgang til prosjektet som ligger på GitHub servere, og dra nytte av Git sin versjonskontroll-teknologi. GitHub har over 20 millioner utviklere, og de fleste av prosjektene er åpne for alle til å kikke på, bidra til eller lære av.
